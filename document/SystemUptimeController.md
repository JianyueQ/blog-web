# 一、前台监控接口

## 1. 获取格式化的累计运行时长

### URL
```
GET /blog/monitor/uptime/formatted
```

### 请求参数
无

### 响应参数
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": "15天3小时25分钟42秒"
}
```

### 响应字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | Integer |状态码，200表示成功 |
| msg | String | 响应消息 |
| data | String |格式化的累计运行时长，格式为：X天X小时X分钟X秒 |

### 说明
- 该接口返回系统自首次启动以来的累计运行时长
- 时长包含历史运行时间和当前会话运行时间
- 数据通过Redis持久化存储，系统重启后可继续累加
- 返回的格式化字符串会根据实际时长自动调整显示单位
